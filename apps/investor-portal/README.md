# Investor Portal

Vestry-style investor portal. Runs on **port 3002** alongside chg-rehab (3000) and Deal Link (3001).

## Shared schema

This app shares its Prisma schema and `@prisma/client` runtime with
`apps/chg-rehab`. There is exactly one `prisma/schema.prisma` (in chg-rehab)
and one `DATABASE_URL`. Prisma client is generated by chg-rehab's
`db:generate`; this app's `lib/prisma.ts` simply imports `@prisma/client`,
which npm workspaces resolve to chg-rehab's installed copy via the workspace
root's `node_modules`.

There is **no separate `prisma/` folder** here on purpose — keep all schema
changes in `apps/chg-rehab/prisma/` and re-run `db:push` / `db:generate`
from that workspace.

If you change the schema, run:

```sh
npm run db:generate --workspace=apps/chg-rehab    # regenerate Prisma Client
npm run db:push --workspace=apps/chg-rehab        # apply DDL to Supabase
```

## Auth model

- Identity is sourced from the same Supabase project that powers chg-rehab.
- An investor is a Supabase user whose `user_profiles.is_investor` flag is
  `true`. Operators (chg-rehab) have `is_investor = false`.
- The middleware in this app rejects users whose flag is **not** `true` and
  redirects them to chg-rehab `/login`. Conversely, chg-rehab's middleware
  rejects investors and redirects them here.
- The `is_investor` column is added to `user_profiles` by
  `apps/chg-rehab/prisma/seed.ts` (idempotent `ALTER TABLE`).
- Per-request memoised resolution lives in `lib/auth.ts#getCurrentInvestor`.

## Env

Reuses the existing chg-rehab env: `SUPABASE_URL`, `SUPABASE_ANON_KEY`,
`SUPABASE_SERVICE_ROLE_KEY`, `DATABASE_URL`, `SESSION_SECRET`. Optional:
`INVESTOR_PORTAL_BASE_URL` (used by chg-rehab in Phase 2 to mint invite
links), `CHG_REHAB_BASE_URL` (used to redirect operators back to chg-rehab
login).
