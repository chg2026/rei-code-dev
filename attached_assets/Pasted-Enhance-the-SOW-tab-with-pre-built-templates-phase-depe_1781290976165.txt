Enhance the SOW tab with pre-built templates, phase dependencies, acceptance criteria, and labor/materials budget split.

Read these files first:
- apps/chg-rehab/app/rehab/[projectId]/sow/page.tsx
- apps/chg-rehab/components/rehab/SowPhase.tsx
- apps/chg-rehab/prisma/schema.prisma (Phase and SOWSection models)

STEP 1 — Schema (raw SQL + prisma generate, NOT prisma db push)

Run migration script at apps/chg-rehab/scripts/migrate-sow-enhancements.ts:

ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS description TEXT;
ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS "laborBudget" DECIMAL(14,2) NOT NULL DEFAULT 0;
ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS "materialsBudget" DECIMAL(14,2) NOT NULL DEFAULT 0;
ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS dependencies INT[] NOT NULL DEFAULT '{}';
ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS "acceptanceCriteria" TEXT[] NOT NULL DEFAULT '{}';
ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS "assignedContractorId" TEXT;

Update schema.prisma Phase model to include these fields. Run: npx prisma generate

STEP 3 — API: apply template
Create apps/chg-rehab/app/api/rehab/[projectId]/sow/apply-template/route.ts
- POST: body { template: 'full_gut' | 'turnover' }
- If project already has phases, return 400 with message "Project already has phases. Clear them first."
- Create Phase records from the template (use template phase number as the phase number field)
- Return the created phases

STEP 4 — API: update phase enhancements
In apps/chg-rehab/app/api/rehab/[projectId]/phases/[phaseId]/route.ts (create if not exists):
- PATCH: accept description, laborBudget, materialsBudget, dependencies (array of ints), acceptanceCriteria (array of strings), assignedContractorId

STEP 5 — UI: SOW page enhancements

In the SOW page, add a "Use Template" button in the action bar (only shown if no phases exist yet). Clicking opens a small modal with two options: "Full Gut Rehab" and "Turnover Rehab". On select, POST to /api/rehab/[projectId]/sow/apply-template and reload.

In SowPhase.tsx, expand the phase row to show (in an expandable section):
- Description (editable text)
- Dependencies: chips showing phase names this phase depends on (e.g. "After: Framing")
- Acceptance Criteria: checklist of criteria items (add/remove items inline)
- Budget split: two inline inputs — Labor $ and Materials $ — that sum to show Total Budget. When either changes, PATCH the phase.
