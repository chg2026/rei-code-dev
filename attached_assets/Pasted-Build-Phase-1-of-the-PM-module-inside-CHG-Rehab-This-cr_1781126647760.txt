═══════════════════════════════════════════
API ROUTES
═══════════════════════════════════════════

FILE: apps/chg-rehab/app/api/pm/spaces/route.ts
GET — list all spaces for the company
POST — create a new space (body: { name, color?, icon? })
Both require auth. Scope to user.companyId.
On POST, also seed 5 default statuses for the new space:
  { name: "To Do", color: "#6B7280", type: "open", order: 0, isDefault: true }
  { name: "In Progress", color: "#3B82F6", type: "active", order: 1 }
  { name: "In Review", color: "#F59E0B", type: "active", order: 2 }
  { name: "Done", color: "#10B981", type: "done", order: 3 }
  { name: "Cancelled", color: "#EF4444", type: "closed", order: 4 }

FILE: apps/chg-rehab/app/api/pm/spaces/[spaceId]/route.ts
PATCH — update space (name, color, icon)
DELETE — delete space

FILE: apps/chg-rehab/app/api/pm/spaces/[spaceId]/lists/route.ts
GET — list all lists in a space
POST — create list (body: { name, color? })

FILE: apps/chg-rehab/app/api/pm/lists/[listId]/route.ts
PATCH — update list
DELETE — delete list

FILE: apps/chg-rehab/app/api/pm/lists/[listId]/tasks/route.ts
GET — list tasks in a list. Query params: statusId, assigneeId, priority, done=1.
  Include: status, assignees (with user name/initials), subtasks count, tags.
  Return only top-level tasks (parentTaskId IS NULL) unless ?subtasks=1.
POST — create task. Body: { name, description?, statusId?, priority?, dueDate?, startDate?, assigneeIds?: string[] }
  On create, log a PmActivity entry (type: "task_created").

FILE: apps/chg-rehab/app/api/pm/tasks/[taskId]/route.ts
GET — get full task detail. Include: status, assignees, subtasks (with their status+assignees), tags, comments (with author), activity (with user).
PATCH — update task fields (name, description, statusId, priority, dueDate, startDate, taskType, timeEstimate, sprintPoints). Log activity on status change.
DELETE — delete task.

FILE: apps/chg-rehab/app/api/pm/tasks/[taskId]/assignees/route.ts
POST — add assignee (body: { userId })
DELETE — remove assignee (body: { userId })

FILE: apps/chg-rehab/app/api/pm/tasks/[taskId]/comments/route.ts
GET — list comments for a task
POST — create comment (body: { body })

FILE: apps/chg-rehab/app/api/pm/spaces/[spaceId]/statuses/route.ts
GET — list statuses for a space
POST — create custom status (body: { name, color, type, order })

FILE: apps/chg-rehab/components/pm/PmListView.tsx
"use client"
Props: { tasks, statuses, listId, spaceId }

Shows tasks grouped by status. Each group:
- Group header: colored status dot + status name + count + "+ Add task" button
- Task rows (see below)
- Empty group shows only the "+ Add task" row

Task row shows: expand arrow (if has subtasks) | task type icon | task name (clickable) | assignee avatars | due date | priority badge | status chip

State: 
- quickCreate: which group is open for quick task creation
- taskDetailId: which task is open in detail panel
- view: "list" | "board" (toggle in toolbar)

If view === "board", render <PmBoardView> instead.

Include a toolbar above the task list:
  [ List | Board ] toggle | [ + New Task ] button | [ Filter ] | [ Group by Status ]
