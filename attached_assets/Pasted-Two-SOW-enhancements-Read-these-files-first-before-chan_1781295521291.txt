ENHANCEMENT 2 — Start Date + Estimated Days on every phase (basis for Gantt)

STEP A — Schema (raw SQL + prisma generate, NOT prisma db push)
Run migration at apps/chg-rehab/scripts/migrate-phase-schedule.ts:

ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS "plannedStartDate" DATE;
ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS "estimatedDays" INT NOT NULL DEFAULT 0;
ALTER TABLE "Phase" ADD COLUMN IF NOT EXISTS "plannedEndDate" DATE GENERATED ALWAYS AS
  (CASE WHEN "plannedStartDate" IS NOT NULL AND "estimatedDays" > 0
    THEN "plannedStartDate" + ("estimatedDays" || ' days')::INTERVAL
    ELSE NULL END) STORED;

Note: if the generated column syntax fails on this Postgres version, skip the generated column and just store plannedEndDate as a regular column that the API computes and saves.

Add to schema.prisma Phase model:
  plannedStartDate  DateTime?  @db.Date
  estimatedDays     Int        @default(0)
  plannedEndDate    DateTime?  @db.Date

Run: npx prisma generate

STEP B — Update phase PATCH endpoint
In apps/chg-rehab/app/api/rehab/[projectId]/phases/[phaseId]/route.ts
Accept plannedStartDate and estimatedDays in the PATCH body. When saving, compute plannedEndDate = plannedStartDate + estimatedDays days (if both present) and save it.

STEP C — SOW phase row UI
In SowPhase.tsx expanded section, add two new fields:
- Planned Start Date (date picker input)
- Estimated Days (number input with "days" label)
- Show computed "Planned End: [date]" below as read-only (plannedStartDate + estimatedDays)
- On change: debounced PATCH to /api/rehab/[projectId]/phases/[phaseId]

Also add these two fields to the "Custom Phase" add form from Enhancement 1.

STEP D — Gantt data readiness
In the Gantt view (apps/chg-rehab/app/rehab/[projectId]/schedule/page.tsx or the Gantt component), ensure it reads plannedStartDate, estimatedDays, and plannedEndDate from each phase for positioning bars. If the Gantt currently uses different fields, migrate it to use these. Each bar should show: phase name, start date, end date, and a progress fill based on phase.status (NotStarted=0%, Active=50%, Complete=100%).
