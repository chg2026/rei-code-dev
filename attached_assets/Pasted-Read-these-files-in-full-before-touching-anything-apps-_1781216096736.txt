--- CHANGE 1: TaskDetailPanel — add "create mode" ---

TaskDetailPanel currently requires an existing taskId and loads its data. Add a create mode:

New props interface:
  mode: "create" | "edit"
  // create mode props:
  linkType?: string      // "property" | "deal" | null
  linkId?: string
  linkLabel?: string
  defaultStatus?: string
  // edit mode props (existing):
  taskId?: string

In create mode:
- Show the same panel layout but with empty/default values
- Title field is a focused input at the top
- Priority defaults to "Medium", Status to "To Do" (if applicable)
- A "Create task" button at the bottom (instead of auto-save)
- On submit: POST to /api/workspace/tasks with { title, priority, dueDate, assigneeId, linkType, linkId, description }
- After successful create: call onCreated(newTaskId) prop, then switch to edit mode with the new ID

In edit mode (existing behavior):
- Load task by taskId, auto-save on blur

Also add to the panel:
- A "Description" textarea section (above Notes if Notes exists, or replace Notes with Description)
- A simple Comments section: fetch GET /api/workspace/tasks/[id]/comments, show list of comments, textarea + Send button that POSTs to same endpoint
  - First check if this API route exists. If not, create it: apps/chg-rehab/app/api/workspace/tasks/[id]/comments/route.ts
  - Check prisma schema for wsTaskComment table. Run: cat apps/chg-rehab/prisma/schema.prisma | grep -i comment
  - If no wsTaskComment model exists, add it to schema.prisma:
      model WsTaskComment {
        id        String   @id @default(cuid())
        taskId    String
        userId    String
        body      String

    And add comments WsTaskComment[] to WsTask model.
    Then run: npx prisma db push --accept-data-loss
