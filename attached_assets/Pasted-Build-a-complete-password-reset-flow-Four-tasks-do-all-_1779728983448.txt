Task 2 — Create /forgot-password page
Create app/forgot-password/page.tsx (server component, force-dynamic, redirects to / if already logged in) and app/forgot-password/ForgotPasswordClient.tsx (client component).
ForgotPasswordClient.tsx must have:

Two tabs: 'Via Email' and 'Via Phone'
Email tab: input for email, submit calls supabase.auth.resetPasswordForEmail(email, { redirectTo: window.location.origin + '/reset-password' }), shows success state 'Check your email — we sent a reset link to [email]'
Phone tab: step 1 — phone number input, calls supabase.auth.signInWithOtp({ phone }), step 2 — OTP input + new password input, calls supabase.auth.verifyOtp({ phone, token, type: 'sms' }) then supabase.auth.updateUser({ password: newPassword })
Back to sign in link at bottom
Use the same CSS class names and card structure as LoginClient.tsx (login-shell, login-card, login-brand, login-logo, login-title, login-sub, login-label, login-input, login-cta, login-foot) so it matches the existing login design exactly
Import getSupabaseBrowserClient from @/lib/supabaseBrowser

Task 3 — Create /reset-password page
Create app/reset-password/page.tsx and app/reset-password/ResetPasswordClient.tsx.
ResetPasswordClient.tsx must:

Show a 'Set new password' form with new password + confirm password fields
On submit: validate passwords match and are at least 8 chars, call supabase.auth.updateUser({ password: newPassword })
On success: show a green checkmark and message 'Password updated — redirecting to sign in' then window.location.href = '/login' after 2 seconds
Use the same login-shell/login-card CSS structure
